WebSocket API 문서

모든 메시지는 { "event": "...", "data": { ... } } 형태의 JSON으로 송수신합니다.

ws://<host>/ws/room?roomId=<UUID>&nickname=<닉네임>
💡 Keep-Alive (Ping/Pong) 안내:
서버 인프라(Render)의 유휴 연결 자동 종료를 방지하기 위해, 클라이언트는 40초에 한 번씩 아무 의미 없는 텍스트(예: "ping")를 서버로 보내거나, WebSocket 내장 Ping/Pong 프레임을 활용해 연결을 유지해야 합니다.

게임 흐름 요약

접속 → room:state (본인) + player:joined (전체)
WAITING: 레디/설정 변경 → 모두 레디 시 READY
READY: 호스트 game:start → game:countdown 3→0 → question:start + question:media
IN_GAME: game:answer → 정답 시 question:correct (3초 후) or 타임아웃 → question:end → 반복 or game:result
RESULT: 호스트 game:restart → WAITING 초기화
클라이언트 → 서버
player player:ready 레디 상태 토글 (비-호스트 전용) ▶
Payload
"event": "player:ready",
"data": { "isReady": true }
모든 비-호스트 접속자가 레디이면 방 상태가 READY로 전환됩니다. 2명 이상 필요.
host host:kick 플레이어 강퇴 (호스트 전용) ▶
Payload
"event": "host:kick",
"data": { "targetNickname": "강퇴할닉네임" }
강퇴 대상에게 player:kicked 전송 후 세션 강제 종료. 나머지에게 player:left 브로드캐스트.
host host:settings_update 방 설정 변경 (WAITING·READY 상태에서만) ▶
Payload
"event": "host:settings_update",
"data": {
  "settings": {
    "questionCount": 10,    // 5~20, 퀴즈 최대 문제 수 이하
    "timeLimit": 20,        // 10~30 (초)
    "wrongAnswerLimit": null, // null = 무제한
    "showAnswer": true
  }
}
READY 상태에서 변경 시 WAITING으로 롤백, 모든 플레이어 레디 해제됩니다.
host host:disband 방 해산 (호스트 전용) ▶
Payload
"event": "host:disband",
"data": {}
전체에게 room:disbanded 전송 후 Redis 키 삭제, 모든 세션 종료.
host host:change 호스트 권한 수동 이양 ▶
Payload
"event": "host:change",
"data": { "targetNickname": "새로운호스트" }
방 상태 WAITING으로 변경, 모두 레디 해제됩니다.
player player:left 자발적 퇴장 ▶
Payload
"event": "player:left",
"data": {}
IN_GAME 중 퇴장 후 접속자 1명 이하이면 NOT_ENOUGH_PLAYERS 에러와 함께 게임 강제 종료.
game game:start 게임 시작 요청 (호스트, READY 상태) ▶
Payload
"event": "game:start",
"data": {}
성공 시 game:countdown(3→2→1→0) 후 question:start + question:media 순서로 전송.
game game:answer 정답 제출 (초당 5회 Rate Limit) ▶
Payload
"event": "game:answer",
"data": { "answer": "정답문자열" }
정답 판정: 소문자 변환 + 앞뒤 공백 제거 + 한글·영소문자·숫자·공백 외 특수문자 제거 후 비교. 다중 정답 중 하나만 맞아도 정답. 오답 한도 초과 시 answer:locked 수신.
question question:playback_error YouTube 재생 오류 보고 → 문제 스킵 ▶
Payload
"event": "question:playback_error",
"data": {
  "questionIndex": 3,   // 현재 문제 번호 (1-based)
  "errorCode": 150     // YouTube Player API 에러 코드
}
서버는 해당 문제를 스킵하고 question:skip을 전체 브로드캐스트합니다.
game game:restart 게임 재시작 (호스트, RESULT 상태) ▶
Payload
"event": "game:restart",
"data": {}
WAITING 초기화 + 점수·진행 데이터 리셋 후 room:state + room:players_updated 전체 브로드캐스트.
서버 → 클라이언트
room room:state 방 전체 상태 스냅샷 ▶
Payload
"event": "room:state",
"data": {
  "roomId": "uuid",
  "roomCode": "ABC123",
  "status": "WAITING",    // WAITING | READY | IN_GAME | RESULT
  "hostNickname": "호스트명",
  "players": [
    { "nickname": "닉네임", "isHost": false, "isReady": true, "isConnected": true, "score": 0 }
  ],
  "settings": { /* RoomSettings */ },
  "gameProgress": null    // IN_GAME 상태일 때만 포함
}
접속 직후 본인에게만 전송. game:restart 후에는 전체 브로드캐스트.
room room:players_updated 플레이어 목록·방 상태 변경 브로드캐스트 ▶
Payload
"event": "room:players_updated",
"data": {
  "status": "READY",
  "players": [ /* Participant[] */ ]
}
room room:settings_updated 방 설정 변경 브로드캐스트 ▶
Payload
"event": "room:settings_updated",
"data": {
  "settings": {
    "questionCount": 10, "timeLimit": 20,
    "wrongAnswerLimit": null, "showAnswer": true
  }
}
room room:disbanded 방 해산 알림 (수신 후 연결 종료) ▶
Payload
"event": "room:disbanded",
"data": { "message": "방장이 방을 해산했습니다." }
player player:joined 신규 플레이어 입장 ▶
Payload
"event": "player:joined",
"data": { "nickname": "신규닉네임", "playerCount": 3 }
player player:reconnected 플레이어 재접속 ▶
Payload
"event": "player:reconnected",
"data": { "nickname": "재접속닉네임" }
30초 이내 재접속 또는 RESULT 상태에서는 기존 데이터를 유지하며 복귀.
player player:disconnected 비정상 연결 끊김 알림 ▶
Payload
"event": "player:disconnected",
"data": { "nickname": "튕긴닉네임", "reconnectTimeoutSec": 30 }
30초 안에 재접속 없으면 서버가 자동 퇴장 처리합니다.
player player:left 퇴장·강퇴·타임아웃 브로드캐스트 ▶
Payload
"event": "player:left",
"data": { "nickname": "나간닉네임", "playerCount": 2 }
player player:kicked 강퇴된 본인에게만 전송 ▶
Payload
"event": "player:kicked",
"data": { "message": "방에서 강퇴되었습니다" }
수신 직후 세션이 강제 종료됩니다.
host host:changed 호스트 변경 브로드캐스트 ▶
Payload
"event": "host:changed",
"data": { "newHostNickname": "새방장닉네임" }
수동 이양·호스트 튕김·퇴장·타임아웃 강퇴 시 모두 발생합니다.
game game:countdown 게임 시작 카운트다운 (1초 간격) ▶
Payload
"event": "game:countdown",
"data": { "count": 3 }  // 3 → 2 → 1 → 0
count 0 직후 question:start가 전송됩니다.
question question:start 문제 시작 정보 (videoId 없음) ▶
Payload
"event": "question:start",
"data": {
  "index": 1,          // 1-based
  "totalCount": 10,
  "timeLimit": 20,    // 초
  "hint": "힌트텍스트"  // null 가능
}
question question:media 재생 미디어 정보 (question:start 직후) ▶
Payload
"event": "question:media",
            "data": {
              "videoId": "dQw4w9WgXcQ",
              "startSec": 30.0,
              "endSec": 60.0
              nextVideoId: abc123xyz
            }
question question:correct 정답자 발생 브로드캐스트 ▶
Payload
"event": "question:correct",
"data": {
  "nickname": "정답자닉네임",
  "score": 1350,      // 기본 1000 + 속도 보너스(최대 500)
  "timeLeft": 14     // 남은 초
}
수신 3초 후 question:end가 옵니다.
question question:end 문제 종료 및 중간 스코어보드 ▶
Payload
"event": "question:end",
"data": {
  "correctAnswer": "정답",   // showAnswer=false 면 null
  "scores": [
    { "nickname": "닉네임", "questionScore": 1350, "totalScore": 2700 }
  ]
}
마지막 문제가 아니면 5초 후 game:countdown 재개. 마지막이면 game:result로 전환.
question question:skip 재생 오류로 문제 스킵 ▶
Payload
"event": "question:skip",
"data": {
  "questionIndex": 3,
  "reason": "재생 오류로 인해 이 문제를 건너뜁니다.",
  "nextIn": 3
}
answer answer:locked 오답 한도 초과 (본인에게만) ▶
Payload
"event": "answer:locked",
"data": { "questionIndex": 3, "message": "이미 오답을 사용했습니다" }
game game:result 최종 결과 (개인화, 본인에게만) ▶
Payload
"event": "game:result",
"data": {
  "ranking": [
    { "nickname": "1등", "totalScore": 8500, "rank": 1, "isMe": true }
  ],
  "myQuestions": [
    { "index": 1, "isCorrect": true, "score": 1350, "speedBonus": 350, "correctAnswer": "정답" }
  ]
}
동점 시 speedBonus 총합으로 순위를 가립니다.
error error 에러 이벤트 ▶
Payload
"event": "error",
"data": { "errorCode": "ROOM_NOT_FOUND", "message": "..." }
errorCode설명연결 종료
ROOM_NOT_FOUND해당 방 없음✓
NICKNAME_TAKEN닉네임 중복 (클라이언트가 #2, #3 등 접미사 붙여 최대 5회 자동 재연결 시도해야 함)✓
GAME_IN_PROGRESS게임 중 신규 입장 불가✓
ROOM_FULL방 인원 초과 (최대 8명)✓
UNAUTHORIZED호스트 권한 없음✗
INVALID_SETTINGS유효하지 않은 설정값✗
EXCEEDS_MAX_QUESTIONS문제 수가 퀴즈 최대치 초과✗
INVALID_STATE현재 상태에서 불가능한 요청✗
NOT_ENOUGH_PLAYERS2명 미만으로 게임 강제 종료✗
NO_QUESTIONS퀴즈에 등록된 문제 없음✗
RATE_LIMITED요청 횟수 제한 초과 (방 입장 60초 10회, 정답 초당 5회)✓(입장시) / ✗(정답시)
